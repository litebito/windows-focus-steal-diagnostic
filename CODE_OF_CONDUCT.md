# Code of Conduct

## Our standards

This is a technical research project. We expect everyone interacting with this repository — issue authors, PR submitters, reviewers, and maintainers — to follow these basic principles:

### Be technically rigorous

- Stick to **factual, reproducible technical findings**. The strength of this project is that anyone can rerun the diagnostic tools on their own system and verify the results.
- Quote sources when referencing external information (community threads, documentation, etc.).
- Don't speculate about motive or intent — describe what the data shows. *"The data indicates X"* is acceptable; *"The vendor is deliberately doing Y to harm users"* is not.

### Be respectful

- Treat all contributors and maintainers with respect, even if you disagree with their conclusions.
- Critique technical claims, not people. *"This analysis seems incorrect because…"* not *"You're wrong"*.
- Don't dismiss other people's reproductions or environments — different setups produce different data, and that's useful.

### Be honest

- Disclose affiliations (see [CONTRIBUTING.md](CONTRIBUTING.md)).
- Don't submit fabricated logs. Submitted log data is expected to be the genuine output of the diagnostic tools running on a real system.
- If you make a mistake, acknowledge it. Mistakes happen; cover-ups make things worse.

### Respect privacy

- Never publish another person's personal information (real name, email, address, license keys, photos, etc.) without their explicit consent.
- If you spot accidental PII in any submission (yours or someone else's), contact a maintainer privately so it can be redacted.
- Do not attempt to deanonymize contributors who are using GitHub usernames or pseudonyms.

### No harassment

- No personal attacks, slurs, threats, or sustained disruption.
- No off-topic political, religious, or commercial promotion.
- No spam, no bot-driven content, no recruitment for unrelated causes.

## Scope

This Code of Conduct applies in all spaces associated with this project: GitHub issues, pull requests, commits, comments, discussions, and any related public-facing channels (cross-posted Reddit/Norton Community threads where this repo is referenced).

## Enforcement

Maintainers may, at their discretion:

- Close issues or PRs that violate this Code
- Lock comment threads on discussions that have become unproductive
- Remove individual comments containing PII, abuse, or off-topic content
- Block users who repeatedly violate this Code from interacting with the repository
- Force-rewrite Git history to remove leaked PII or harmful content

If you experience or witness behavior that violates this Code, please contact a maintainer by opening a [private GitHub security advisory](https://github.com/litebito/windows-focus-steal-diagnostic/security/advisories/new) on this repository.

## Acknowledgement

This Code of Conduct is loosely inspired by the [Contributor Covenant](https://www.contributor-covenant.org/) but has been kept short and focused on what's actually relevant to a small technical research project.
